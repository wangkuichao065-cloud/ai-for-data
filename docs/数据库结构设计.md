# 数据分析系统综合实践 — 数据库结构设计

> 技术栈：Neo4j（知识图谱） + MySQL（业务数据） + FAISS（向量检索）

---

## 一、数据库总体架构

本系统采用三库协同设计，各司其职：

```
┌─────────────────────────────────────────────────────┐
│                    FastAPI 后端                      │
├──────────┬──────────────┬──────────────────────────┤
│          │              │                          │
│  Neo4j   │    MySQL     │        FAISS            │
│          │              │                          │
│ 知识图谱 │   业务数据    │     向量检索              │
│          │              │                          │
│ 课程知识 │ 用户/会话/   │ 文档分块Embedding        │
│ 节点关系 │ 反馈/文件记录 │ 相似度检索               │
│          │              │                          │
└──────────┴──────────────┴──────────────────────────┘
```

| 数据库 | 职责 | 数据特征 |
|--------|------|----------|
| Neo4j | 课程知识点图谱建模 | 节点 + 关系，图结构 |
| MySQL | 用户、会话、文件等业务数据 | 结构化关系型数据 |
| FAISS | 文档向量索引与相似度检索 | 高维浮点向量 |

### 2.3 关系类型定义

共定义 6 类关系：

#### 2.3.1 CONTAINS（包含）

课程 → 章节 → 知识点的层级包含关系。

```cypher
(:Course)-[:CONTAINS]->(:Chapter)
(:Chapter)-[:CONTAINS]->(:KnowledgePoint)
(:KnowledgePoint)-[:CONTAINS]->(:KnowledgePoint)
```

| 属性 | 类型 | 说明 |
|------|------|------|
| order | int | 在父节点下的顺序 |

#### 2.3.2 DEPENDS_ON（依赖）

知识点之间的前置依赖关系。

```cypher
(:KnowledgePoint)-[:DEPENDS_ON]->(:KnowledgePoint)
(:Algorithm)-[:DEPENDS_ON]->(:KnowledgePoint)
```

| 属性 | 类型 | 说明 |
|------|------|------|
| strength | string | 依赖强度：strong / medium / weak |

#### 2.3.3 APPLIED_TO（应用于）

算法/知识点 → 应用场景。

```cypher
(:Algorithm)-[:APPLIED_TO]->(:Application)
(:KnowledgePoint)-[:APPLIED_TO]->(:Application)
```

#### 2.3.4 SIMILAR_TO（相似于）

知识点/算法之间的相似关系，用于推荐。

```cypher
(:KnowledgePoint)-[:SIMILAR_TO]->(:KnowledgePoint)
(:Algorithm)-[:SIMILAR_TO]->(:Algorithm)
```

| 属性 | 类型 | 说明 |
|------|------|------|
| similarity | float | 相似度 0.0-1.0 |
| dimension | string | 相似维度：原理/应用/数学基础 |

#### 2.3.5 IMPLEMENTS（实现）

知识点 → 实现该知识点的算法。

```cypher
(:KnowledgePoint)-[:IMPLEMENTS]->(:Algorithm)
```

#### 2.3.6 BELONGS_TO（属于）

知识点所属课程。

```cypher
(:KnowledgePoint)-[:BELONGS_TO]->(:Course)
(:Algorithm)-[:BELONGS_TO]->(:Course)
```

### 2.6 常用查询 Cypher

```cypher
// 查询某课程的完整知识树
MATCH path = (c:Course {node_id: 'course_ml'})-[:CONTAINS*]->(n)
RETURN path;

// 查询某知识点的前置依赖
MATCH (kp:KnowledgePoint {node_id: 'kp_svm'})-[:DEPENDS_ON*]->(dep:KnowledgePoint)
RETURN dep.name AS prerequisite;

// 查询相似知识点
MATCH (kp:KnowledgePoint {node_id: 'kp_lr'})-[r:SIMILAR_TO]->(similar:KnowledgePoint)
RETURN similar.name, r.similarity
ORDER BY r.similarity DESC;

// 查询某知识点的完整上下文（用于RAG增强）
MATCH (kp:KnowledgePoint {name: '支持向量机'})
OPTIONAL MATCH (kp)-[:DEPENDS_ON]->(dep:KnowledgePoint)
OPTIONAL MATCH (kp)-[:SIMILAR_TO]->(sim:KnowledgePoint)
OPTIONAL MATCH (kp)-[:IMPLEMENTS]->(algo:Algorithm)
OPTIONAL MATCH (kp)-[:APPLIED_TO]->(app:Application)
RETURN kp, collect(DISTINCT dep) AS dependencies, collect(DISTINCT sim) AS similar, collect(DISTINCT algo) AS algorithms, collect(DISTINCT app) AS applications;

// 查询学习路径（最短路径）
MATCH path = shortestPath(
  (start:KnowledgePoint {node_id: 'kp_lr'})
  -[:DEPENDS_ON|CONTAINS*0..5]->
  (end:KnowledgePoint {node_id: 'kp_svm'})
)
RETURN path;
```

## 四、FAISS 向量库设计

### 4.1 向量库信息

```
索引类型：FAISS IndexFlatIP（内积相似度，配合归一化向量）
向量维度：512（bge-small-zh-v1.5 模型）
Embedding模型：BAAI/bge-small-zh-v1.5
索引存储路径：data/faiss_index/
```

### 4.2 向量数据结构

FAISS 本身只存储向量数组，元数据通过 MySQL 的 `file_chunks` 表维护映射关系。

```
FAISS 向量索引 (index.faiss)
┌─────────────────────────────────────────────────┐
│  Index 0: [0.012, -0.034, 0.056, ..., 0.078]  │ ← chunk_id=1
│  Index 1: [0.045, 0.023, -0.011, ..., 0.034] │ ← chunk_id=2
│  Index 2: [-0.032, 0.067, 0.089, ..., 0.012]  │ ← chunk_id=3
│  ...                                            │
│  Index N: [0.056, -0.045, 0.023, ..., 0.067]  │ ← chunk_id=N
└─────────────────────────────────────────────────┘
                    ↕ 映射
MySQL file_chunks 表
┌──────────┬─────────┬──────────┬────────────┬───────────┐
│ chunk_id │ file_id │ content  │ start_page │faiss_index│
├──────────┼─────────┼──────────┼────────────┼───────────┤
│ 1        │ file_01 │ "SVM..." │ 45         │ 0         │
│ 2        │ file_01 │ "核函数.."│ 46         │ 1         │
│ 3        │ file_02 │ "K-Means"│ 120        │ 2         │
│ ...      │ ...     │ ...      │ ...        │ ...       │
└──────────┴─────────┴──────────┴────────────┴───────────┘
```

### 4.3 向量检索流程

```
用户问题
    │
    ▼
Embedding 模型 (bge-small-zh-v1.5)
    │
    ▼
问题向量 [512维]
    │
    ▼
FAISS 搜索 (IndexFlatIP, top_k=5)
    │
    ▼
返回 Index [0, 3, 7, 12, 15] + 相似度分数
    │
    ▼
MySQL 查询 file_chunks WHERE faiss_index IN (0, 3, 7, 12, 15)
    │
    ▼
获取分块内容 + 来源文件信息
    │
    ▼
拼接 Prompt 上下文 → DeepSeek 生成回答
```

### 4.4 FAISS 管理操作

```python
# 向量入库流程
def add_document_chunks(file_id: str, chunks: list[str]):
    """文档分块 → Embedding → FAISS 入库"""
    embeddings = embedding_model.encode(chunks)  # [N, 512]
    faiss.normalize_L2(embeddings)               # 归一化
    start_index = faiss_index.ntotal              # 当前索引起点
    faiss_index.add(embeddings)                   # 入库
    # 记录映射关系到 MySQL
    for i, chunk in enumerate(chunks):
        db.execute(
            "INSERT INTO file_chunks (file_id, chunk_index, content, faiss_index) VALUES (?, ?, ?, ?)",
            (file_id, i, chunk, start_index + i)
        )
    faiss.write_index(faiss_index, INDEX_PATH)   # 持久化

# 向量检索流程
def search_similar(query: str, top_k: int = 5):
    """问题 → 向量检索 → 返回相关分块"""
    query_vec = embedding_model.encode([query])   # [1, 512]
    faiss.normalize_L2(query_vec)
    scores, indices = faiss_index.search(query_vec, top_k)
    # 查询 MySQL 获取内容
    results = db.execute(
        "SELECT chunk_id, file_id, content, start_page FROM file_chunks WHERE faiss_index IN ?",
        (indices[0].tolist(),)
    )
    return [{"content": r.content, "score": float(scores[0][i]),
             "source": r.file_id, "page": r.start_page}
            for i, r in enumerate(results)]
```
