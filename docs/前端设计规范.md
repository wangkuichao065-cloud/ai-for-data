## 课程数据分析智能平台 — 前端设计规范

> 基于知识图谱与大模型的智能学习系统 | Vue 3 + Element Plus + ECharts

---

### 一、设计概述

本平台面向高校学生与教师，整合知识图谱可视化、大模型 RAG 智能问答与 AI 数字教师三大核心能力。前端设计追求「学术感 + 科技感」的融合风格：以深靛蓝-紫色渐变构建品牌识别，以白底圆角卡片承载内容，以柔和阴影与充裕留白保证阅读舒适度。整体视觉语言参考现代 SaaS 仪表盘范式，同时融入教育场景特有的温暖感与知识图谱的网状视觉意象。

**设计原则：**
信息密度适中，一屏内尽量呈现完整业务上下文；操作路径清晰，核心功能两步可达；动效克制但有节奏感，避免过度装饰；色彩服务于语义，不同颜色承载明确的功能含义。

### 四、布局架构

#### 4.1 整体结构

采用经典左侧导航 + 顶部栏 + 主内容区三栏布局，全屏高度 `100vh`，无页面级滚动。

```
┌──────────────────────────────────────────────────┐
│ Sidebar (220px / 64px折叠) │ Header (60px)       │
│                            │──────────────────── │
│  Logo (60px高)             │                     │
│  ─────────────             │   Main Content      │
│  菜单项 × 7                │   padding: 20px     │
│  (每项44px高, 圆角8px)      │   overflow-y: auto  │
│                            │   bg: #f5f7fa       │
│  ─────────────             │                     │
│  折叠按钮                   │                     │
└──────────────────────────────────────────────────┘
```

**CSS 变量：**
- `--sidebar-width: 220px`（展开态）
- `--sidebar-collapsed: 64px`（折叠态）
- `--header-height: 60px`

#### 4.2 侧边栏

**视觉风格：** 深靛蓝纵向三段渐变 `linear-gradient(180deg, #1e1b4b 0%, #312e81 50%, #3730a3 100%)`，营造沉稳的科技氛围。

**结构：** 顶部 Logo 区（高度 60px，底部白色 8% 透明度分割线）→ 中部菜单区（flex:1, 可滚动, 上下 padding 8px）→ 底部折叠按钮区（padding 12px，顶部分割线）。

**菜单项规范：**
- 高度 44px，行高 44px，圆角 8px，间距 4px
- 默认态：文字 `rgba(255,255,255,0.65)`，图标同色
- 悬停态：背景 `rgba(255,255,255,0.08)`
- 选中态：背景 `rgba(255,255,255,0.15)`，文字白色，字重 600
- 图标使用 Element Plus Icons：Odometer、Share、ChatDotRound、UserFilled、TrendCharts、FolderOpened、Setting

**七个导航项：**
仪表盘 `/` → 知识图谱 `/knowledge-graph` → 智能问答 `/qa` → 数字教师 `/teacher` → 数据分析 `/analysis` → 文件管理 `/files` → 系统管理 `/system`

#### 4.3 顶部栏

高度 60px，白底，底部 1px `#e2e8f0` 分割线 + 极淡投影 `0 1px 3px rgba(0,0,0,0.04)`。

左侧显示当前页面标题（18px/600）。右侧依次为：搜索图标、全屏切换、通知铃铛（红色角标 `#ef4444`，显示未读公告数）、用户信息组（圆形渐变头像 32px + 用户名 + 下拉箭头）。用户信息区悬停时背景变灰 `var(--bg)`，圆角 8px。

#### 4.4 主内容区

padding 20px，背景 `#f5f7fa`，独立纵向滚动（`overflow-y: auto`），高度 `calc(100vh - 60px)`。

### 五、通用组件规范

#### 5.1 卡片 (Card)

- 背景白色，圆角 12px，内边距 20px
- 投影：默认 `0 2px 12px rgba(0,0,0,0.06)`，悬停 `0 8px 30px rgba(0,0,0,0.08)`
- 悬停过渡 `all 0.3s cubic-bezier(0.4, 0, 0.2, 1)`
- 卡片标题：15px/600，左侧竖线装饰（宽 4px、高 16px、品牌渐变色、圆角 2px）

#### 5.2 按钮

- 主操作按钮：品牌渐变背景，无边框，高度 40-44px，圆角 10-12px，白色 600 字重文字
- 次操作按钮：使用 Element Plus 默认样式
- 文字按钮：无边框无背景，用于反馈、下载等操作列

#### 5.3 表格

- 圆角 8px，溢出隐藏
- 表头背景 `#f5f7fa`，字重 600
- 支持斑马纹 `stripe`

#### 5.4 表单

- 输入框圆角 12px，聚焦时边框变为 `#6366f1` + `0 0 0 2px rgba(99,102,241,0.1)` 外发光
- 标签宽度：系统配置页 200px，个人设置 100px
- 滑块、数字输入、开关等使用 Element Plus 默认样式

#### 5.5 滚动条

全局自定义：宽/高 6px，透明轨道，圆角滑块 `#d1d5db`，悬停 `#9ca3af`。

#### 5.6 过渡动效

统一使用 `all 0.3s cubic-bezier(0.4, 0, 0.2, 1)` 作为缓动曲线。Vue 路由切换使用 `fade` 过渡（opacity 0→1，200ms ease）。打字指示器动画：三个 6px 圆点上下跳动，1.4s 循环，间隔 0.2s 错开。流式输出光标闪烁：1s 循环。状态点呼吸灯：2s 循环 opacity 变化。

### 六、各页面详细设计

#### 6.1 登录/注册页

**整体氛围：** 全屏沉浸式深靛蓝渐变背景 `linear-gradient(135deg, #1e1b4b, #312e81, #4f46e5, #6366f1)`，叠加两个半透明装饰圆（右上角 600px、左下角 400px，白色 3-4% 透明度）。

**登录卡片：** 居中定位，宽 440px（最大 90vw），半透明白底 `rgba(255,255,255,0.95)` + 毛玻璃 `backdrop-filter: blur(20px)`，圆角 20px，内边距 40px，深投影 `0 20px 60px rgba(0,0,0,0.3)`。

**卡片结构：**
- 顶部：🧠 Emoji Logo（48px）+ 标题「课程数据分析平台」（22px/700）+ 副标题「基于知识图谱与大模型的智能学习系统」（14px/#94a3b8）
- 中部：Element Plus Tabs（登录/注册切换），表单项包含用户名、密码、邮箱（注册时）、角色选择（学生/教师单选）
- 底部：渐变全宽按钮（44px 高），下方「还没有账号？立即注册」链接

**路由守卫：** 已登录用户访问登录页自动跳转首页。

#### 6.2 仪表盘页面

仪表盘是系统的核心数据概览界面，采用「KPI 指标 → 趋势图表 → 热门排行」的信息层级从上到下排布。

**第一行：KPI 卡片组**
使用 CSS Grid `repeat(auto-fit, minmax(200px, 1fr))`，gap 16px，底部间距 20px。共 6 张指标卡：

| 指标 | 图标 | 色系 | 数值格式 | 变化描述 |
|------|------|------|----------|----------|
| 总提问数 | 💬 | 紫色 | 整数 | +N 本周 / 上升箭头绿色 |
| 今日提问 | 📝 | 蓝色 | 整数 | +N 较昨日 |
| 总用户数 | 👥 | 绿色 | 整数 | +N 本周 |
| 活跃用户 | 🔥 | 橙色 | 整数 | N% 活跃率 |
| 知识覆盖率 | 📚 | 粉色 | 百分比 | +N% 本月 |
| 平均满意度 | ⭐ | 青色 | 一位小数 | N.N / 5.0 |

每张卡片悬停时向上位移 2px + 加深投影。

**第二行：双列图表区**
Grid `repeat(auto-fit, minmax(400px, 1fr))`，gap 20px：
- **左：提问趋势折线图**（近7天）—— 单系列平滑曲线，线宽 3px 靛蓝色，圆形数据点 8px 带白色描边，面积填充从上 25% 透明度渐变到 2%。X 轴日期标签（#94a3b8/11px），Y 轴虚线网格。
- **右：课程提问分布饼图** —— 环形图（内径 45%、外径 70%），两色：靛蓝（机器学习）+ 天蓝（数据挖掘），圆角扇区 + 白色 2px 描边，底部图例。

**第三行：热门知识点进度条**
单卡片全宽。每个条目结构：左标签（80px 固定宽，13px 次色文字）→ 中间进度条（flex:1，高 8px，灰色底 `#f5f7fa`，渐变填充，圆角 4px，1s ease 动画）→ 右百分比（40px，右对齐，13px/600）。共 5 条，间距 14px。

**加载态：** 使用 Element Plus Skeleton（6 行，带动画）占位。

#### 6.3 知识图谱页面

知识图谱是平台的视觉核心，采用「工具栏 → 全屏力导向图 → 侧滑详情面板」的交互结构。

**工具栏：**
水平弹性布局，间距 12px，可换行。从左到右：课程筛选下拉框（160px）→ 搜索输入框（240px）→ 搜索按钮（渐变背景）→ 弹性空白 → 类别图例标签组（5 个圆角小标签，effect="dark"，无边框白色文字）。

五个节点类别及颜色：章节 `#6366f1`、概念 `#3b82f6`、算法 `#22c55e`、技术 `#f97316`、应用 `#ec4899`。

**图谱画布：**
白色卡片容器，圆角 12px，投影。内部 ECharts 力导向图，高度 `calc(100vh - 220px)`。

力导向参数：斥力 280，引力 0.08，边长范围 80-200px，摩擦系数 0.6。节点根据类别着色，symbolSize 按重要性分级（50/45/42/40/35/30/28/26/24/22/20），大于等于 30 的节点显示标签（11px，#1a1a2e）。

边类型与颜色映射：CONTAINS `#6366f1`、BELONGS_TO `#3b82f6`、EXTENDS/BASED_ON `#22c55e`、USES `#f97316`、APPLIES_TO `#ec4899`、PREREQ `#8b5cf6`、RELATED_TO/EXPLAINS `#94a3b8`。线宽 1.5px，曲度 0.15，箭头指向目标节点。

交互：支持拖拽节点、画布漫游（roam: true）、悬停聚焦邻接节点（emphasis: adjacency）。

**详情面板（侧滑）：**
绝对定位右侧，宽 340px，全高，白色底，左侧 1px 分割线。默认隐藏在画布外（translateX(100%)），点击节点后面板滑入（transform: translateX(0)，0.3s 过渡）。

面板内容：标题（18px/700）+ 关闭按钮 → 分割线 → 类型标签（圆角药丸，背景色取节点类别色）→ 描述段落（14px，行高 1.8）→ 关联知识点列表（每项：灰色底 `#f5f7fa`，圆角 8px，8px 12px 内边距，🔗图标 + 名称 + 关系类型标签，悬停变浅蓝底 `#eef2ff` + 主色文字）。

#### 6.4 智能问答页面

问答页面采用类 ChatGPT 的「左侧会话列表 + 右侧聊天主区」双栏布局。

**布局：**
CSS Grid `280px 1fr`，gap 16px，高度 `calc(100vh - 140px)`。响应式断点 1200px 以下隐藏会话列表侧栏。

**左侧会话列表：**
白色卡片，顶部「+ 新对话」全宽渐变按钮。下方会话列表可滚动，每项 padding 12px，圆角 8px。选中项背景 `#eef2ff`。显示会话标题（14px/500，单行溢出省略号）和底部元信息（课程名 + 相对时间，12px 弱色文字）。

**右侧聊天主区：**
白色卡片，三段结构：
- **消息区**（flex:1, overflow-y:auto, padding 20px）
  - 空状态：居中图标🤖（48px）+ 欢迎语 + 快捷提问标签组（4 个预设问题，点击即发送）
  - 消息气泡：用户消息右对齐，头像 36px 圆形灰色底「我」字；AI 消息左对齐，头像渐变底「AI」字。气泡 padding 12px 16px，圆角 12px。用户气泡主色底白字，AI 气泡灰色底 `#f5f7fa` 主色文字
  - 引用来源：AI 回答下方分割线后展示，📄图标 + 文件名 + 页码 + 相关度百分比标签
  - 反馈按钮：AI 消息下方「👍 有帮助」「👎 需改进」文字按钮
- **流式加载态：** AI 头像 + 气泡内渲染中的 Markdown 内容 + 闪烁光标 `|`
- **输入区**（padding 16px 20px，顶部 1px 分割线）：两行 textarea（圆角 12px，不可 resize）+ 发送按钮（渐变背景，圆角 10px，高 40px）

#### 6.5 数字教师页面

数字教师页面呈现「左侧教师形象展示 + 右侧对话交互」的对称双栏布局。

**布局：**
CSS Grid `1fr 1fr`，gap 20px，高度 `calc(100vh - 140px)`。1200px 以下变为单栏。

**左侧教师形象区：**
白色卡片，中央垂直排列。背景叠一层柔和渐变 `var(--gradient-soft)` 50% 透明度。

从上到下：
- **头像：** 280px 圆形，品牌渐变底，中央 👩‍🏫 Emoji（100px），深投影 `0 20px 60px rgba(99,102,241,0.2)`
- **姓名：** 「小智老师」，20px/700
- **状态：** 绿色呼吸灯（8px 圆点，2s 循环）+ 「在线」/「离线」文字
- **情绪按钮组：** 四个圆角药丸按钮（平静/开心/思考/鼓励），13px，默认灰边框，选中态主色底白字
- **学习进度卡片：** 嵌套标准卡片，包含三列统计（总提问数/学习时长/当前等级，22px/700 主色数值）+ 已掌握知识点标签组（绿色 small tag）+ 待加强知识点标签组（橙色 small tag）

**图表矩阵（三行两列）：**

#### 6.7 文件管理页面

### 八、交互与状态规范

#### 8.1 全局消息反馈

使用 Element Plus Message 组件，z-index 强制 9999。成功（绿色）、警告（橙色）、错误（红色）、信息（蓝色）四种类型。

#### 8.2 确认操作

删除、重建索引、退出登录等不可逆操作，统一使用 Element Plus MessageBox 确认弹窗，包含确认/取消按钮和警告图标。

#### 8.3 加载状态

- 页面级：Skeleton 骨架屏（带动画）
- 按钮级：loading 旋转图标 + 禁用态
- 表格级：Element Plus v-loading 遮罩
- 流式输出：逐字渲染 + 闪烁光标

#### 8.4 空状态

每个数据区域在无内容时显示居中图标 + 友好提示文案 + 引导操作（如问答页的快捷提问标签）。

#### 8.5 错误处理

API 调用统一 try-catch 捕获，静默失败场景（如页面初始化时非关键数据）不弹消息，关键操作失败弹出红色错误提示。图表和列表在加载失败时保留空状态，不阻塞页面。

### 九、技术实现要点

#### 9.1 技术栈

| 层 | 选型 | CDN 引入 |
|----|------|----------|
| 框架 | Vue 3 (Composition API) | `unpkg.com/vue@3` |
| UI 库 | Element Plus | CDN + 中文语言包 |
| 图标 | @element-plus/icons-vue | CDN 全局注册 |
| 路由 | Vue Router 4 (Hash 模式) | CDN |
| 图表 | ECharts 5 | CDN |
| 图谱 | ECharts Force Graph | 内置于 ECharts |
| Markdown | marked.js + DOMPurify | CDN |
| HTTP | Fetch API + SSE | 原生 |

#### 9.2 文件结构

```
frontend/
├── index.html          # SPA 入口，CDN 引入
├── css/
│   └── style.css       # 全局样式（~1000行）
└── js/
    ├── app.js          # Vue 应用初始化、路由、导航守卫
    ├── api.js          # API 封装层（JWT/SSE/Mock 回退）
    ├── pages.js        # 9 个页面组件定义
    ├── charts.js       # ECharts 图表工厂函数
    └── mock-data.js    # 完整模拟数据集
```

#### 9.3 API 通信规范

- 统一响应格式：`{ code: 200, message: "success", data: {...}, timestamp: "..." }`
- JWT Token 存于 localStorage，通过 Authorization: Bearer 头传递
- SSE 流式传输用于问答和数字教师对话，逐 token 渲染
- Mock 数据回退机制：后端不可用时自动切换本地数据

#### 9.4 8GB 显存约束对前端的影响

前端需展示当前模型加载状态（系统管理页的模型状态卡片），并在切换功能时配合后端的模型交替加载策略。数字教师页面的 Stable Diffusion 形象生成为异步操作，前端展示进度条。
